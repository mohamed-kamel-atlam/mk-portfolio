## Overview

MISRA AI is an AI-powered data-analysis platform — the graduation project I led
as front-end engineer on a team of seven. It lets users upload datasets, run
exploratory analysis, generate AI-driven predictions, detect data drift, and
produce reports, all through a single cohesive interface.

## Problem

The product spanned many features — data ingestion, visualization, model
inference, monitoring, and reporting — built simultaneously by a multi-role
team. Without a clear frontend architecture, that breadth would have collapsed
into coupled code, duplicated state, and merge friction.

## Solution

I owned the complete frontend architecture: a feature-based structure backed by
a reusable component library and design system, a disciplined server/client
state split, and dashboards tuned for rendering performance over large datasets.

## My Role

As Lead Front-End Engineer I set the architecture, built the shared component
library, implemented authentication and role-based access control against the
identity APIs, and integrated the REST and AI/ML service endpoints. I drove the
team's GitHub Flow — pull requests, code reviews — and coordinated delivery in
Jira.

## Architecture Decisions

The structured decisions above (feature-based architecture, server/client state
split, performance-first dashboards) were the load-bearing calls. Each was made
to let a large team move fast without accumulating architectural debt.

## Challenges

Rendering large analytical datasets smoothly was the hardest problem. Profiling
guided targeted memoization, virtualized/lazy rendering, and code splitting so
interaction stayed responsive as data volume grew.

## Lessons Learned

Architecture is a team enabler, not a personal preference: the feature
boundaries and shared design system were what actually let seven people ship in
parallel. Investing in that structure early paid back every sprint.
