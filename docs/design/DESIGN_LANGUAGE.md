# Design Language

---

# Philosophy

The design should communicate engineering quality before showing any project.

The portfolio should feel like a premium AI SaaS application rather than
a personal website.

---

# Design Goals

- Premium
- Modern
- Memorable
- Fast
- Clean
- Accessible
- Minimal
- Elegant

---

# Inspiration

Inspired by products such as:

- Vercel
- Linear
- Huly
- Raycast
- OpenAI

The goal is inspiration, never imitation.

---

# Layout Philosophy

Use generous whitespace.

Allow every section to breathe.

Avoid visual clutter.

Content should guide the user's eyes naturally.

---

# Typography Philosophy

Typography is the primary design element.

Large headings.

Readable body text.

Strong hierarchy.

Consistent spacing.

Minimal font weights.

Typography should communicate confidence.

---

# Color Philosophy

Dark-first.

Light mode should feel equally premium.

Colors should support the content rather than dominate it.

Accent colors should be used intentionally.

Never overuse gradients.

---

# Motion Philosophy

Animations should explain interfaces.

Animations should improve perception.

Animations should improve navigation.

Animations should never distract.

---

# Component Philosophy

Components should be:

Reusable

Accessible

Composable

Predictable

Minimal

Every component must have one responsibility.

---

# Content Philosophy

Content is part of the design.

Avoid unnecessary text.

Every sentence should provide value.

Every heading should communicate purpose.

---

# Interaction Philosophy

Hover states should feel responsive.

Focus states should be visible.

Transitions should be subtle.

Scrolling should feel natural.

Navigation should feel effortless.

---

# Visual Hierarchy

Priority order:

1. Typography

2. Content

3. Spacing

4. Motion

5. Decorative Elements

Decoration should never compete with content.

---

# Accessibility

Accessibility is part of the design.

Not an afterthought.

Support:

Keyboard Navigation

Screen Readers

Reduced Motion

High Contrast

Visible Focus

Semantic HTML

---

# Responsive Design

Design mobile-first.

Scale progressively.

Never hide important information on smaller screens.

---

# Design Principles

Less but better.

Function before decoration.

Purpose before animation.

Engineering before marketing.

Consistency over creativity.

Quality over quantity.

---

# Design Review Checklist

Before shipping any UI ask:

Does this communicate trust?

Does this improve readability?

Is the hierarchy obvious?

Does the animation have a purpose?

Is it accessible?

Is it responsive?

Is it consistent?

Would this exist in a premium SaaS product?

If any answer is "No",

Do not ship.
